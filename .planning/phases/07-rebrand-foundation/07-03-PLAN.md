# 07-03: Domain Setup and Vercel Configuration for lawnbowl.app

## Objective
Configure all code-level domain references, Vercel settings, and security headers
for the production domain `lawnbowl.app`.

## Steps

1. **next.config.ts** -- Add production domain to image remotePatterns, HSTS and DNS prefetch headers
2. **vercel.json** -- Verify redirect rules (lawnbowling.app -> lawnbowl.app, www variants), add HSTS, Permissions-Policy headers
3. **Fix typo** -- clubs/[state]/page.tsx had `lawnbowls.app` instead of `lawnbowl.app`
4. **schema.ts** -- Fix areaServed (Australia -> United States), priceCurrency (AUD -> USD), addressCountry (AU -> US), featureList wording
5. **robots.ts / sitemap.ts** -- Already correctly using `https://lawnbowl.app` (no changes needed)
6. **.env.local.example** -- Add NEXT_PUBLIC_APP_URL, EMAIL_FROM, VAPID key placeholders
7. **Build verification** -- TypeScript passes clean; build fails only on pre-existing Supabase env var issue (not related to our changes)

## Out of Scope
- DNS record creation (requires Vercel dashboard / registrar)
- SSL certificate provisioning (handled automatically by Vercel)
- Actual environment variable values in Vercel dashboard
