   - Show different message and color based on status:

     - `declined`: neutral/gray, "{targetName} declined your request" with X icon
     - `expired`: amber, "Request to {targetName} expired" with clock icon

2. **Update `src/app/board/page.tsx`:**
   The board page already imports and uses RequestModal, IncomingRequest, IncomingRequestProvider, and MatchQueue. Add:

   - Import `useSentRequests` from `@/lib/hooks/useSentRequests`
   - Import `SentRequestToast` from `@/components/partner/SentRequestToast`
   - Add state: `const [statusNotifications, setStatusNotifications] = useState<Array<{id: string; targetName: string; sport: string; status: string}>>([])`
   - Create `handleStatusChange` callback:
     ```typescript
     const handleStatusChange = useCallback((request: PartnerRequest, newStatus: string) => {
       // Find target name from the sentRequests data or from the request
       const notification = {
         id: request.id,
         targetName: request.target?.display_name || request.target?.name || "Player",
         sport: request.sport,
         status: newStatus,
       };
       setStatusNotifications(prev => [...prev, notification]);
     }, []);
     ```
   - Call the hook: `const { sentRequests } = useSentRequests(currentPlayer?.id ?? null, handleStatusChange);`
   - Compute `sentRequestTargetIds`: `const sentRequestTargetIds = new Set(sentRequests.map(r => r.target_id));`
   - Pass `sentRequestTargetIds` down to `AvailabilityBoard` as a prop (or directly to PlayerCard -- see Task 2)
   - Render `SentRequestToast` notifications:
     ```tsx
     {statusNotifications.map(notif => (
       <SentRequestToast
         key={notif.id}
         targetName={notif.targetName}
         sport={notif.sport}
         status={notif.status as "accepted" | "declined" | "expired"}
         onDismiss={() => setStatusNotifications(prev => prev.filter(n => n.id !== notif.id))}
       />
     ))}
     ```
   - Place toast renders inside the `IncomingRequestProvider` so they share the Toast viewport

   Note: The board page already has `handleSendRequest` wired to `/api/partner/request`. If the response includes `status: "auto_matched"`, show a SentRequestToast with "accepted" status immediately (the API now returns this for mutual requests from plan 04-01). Update `handleSendRequest`:
   ```typescript
   const data = await response.json();
   if (data.status === "auto_matched") {
     setStatusNotifications(prev => [...prev, {
       id: crypto.randomUUID(),
       targetName: requestTarget?.name || "Player",
       sport: selectedSport, // you'll need to capture this
       status: "accepted",
     }]);
   }
   ```
   Actually, the current `handleSendRequest` doesn't have access to the sport (it's handled inside RequestModal). The simpler approach: the `useSentRequests` realtime subscription will detect the status change and fire `onStatusChange`. So just rely on the hook for all notifications.
