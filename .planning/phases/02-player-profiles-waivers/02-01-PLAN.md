  <action>
    The scaffolded code is nearly complete. Wire up the remaining gaps:

    1. **src/lib/supabase/middleware.ts** -- Add profile/waiver gating logic to `updateSession()`:
       - After checking if user is authenticated, add profile-existence check
       - Define a list of "setup-allowed" paths that users without profiles can access: `/profile/setup`, `/api/profile`, `/api/profile/avatar`, `/api/waiver/sign`, `/api/insurance/status`, `/auth/callback`, `/login`, `/signup`
       - For performance, do NOT query the database on every request. Instead use a cookie-based approach:
         - Check for cookie `pap_has_profile`. If present, user has a profile -- skip DB check
         - If cookie is missing and path is not in setup-allowed list, query `players` table for user_id. If no row found, redirect to `/profile/setup`
         - If player row found, set `pap_has_profile=1` cookie on the response (path `/`, httpOnly false, maxAge 1 year)
       - Similarly check for `pap_has_waiver` cookie. If user has profile but no waiver cookie and is trying to access `/board` or `/`, redirect to `/profile/setup`
       - Keep existing logic for unauthenticated users (redirect to /login)

    2. **src/app/api/profile/route.ts** -- Add to the existing handlers:
       - Add a `GET` handler that returns the current user's profile by calling `getPlayerByUserId(user.id)`. Return 404 if no profile exists.
       - In the `POST` handler, after successful player creation, set a `pap_has_profile=1` cookie on the NextResponse. Do this by creating the response with `NextResponse.json(player, { status: 201 })` and then calling `response.cookies.set("pap_has_profile", "1", { path: "/", maxAge: 60 * 60 * 24 * 365 })` before returning.
       - Add server-side validation of `skill_level` against the enum `["beginner", "intermediate", "advanced"]`. Return 400 if invalid.
       - Add server-side validation of `sports` array -- each value must be one of `["pickleball", "lawn_bowling", "tennis", "badminton", "table_tennis"]`. Return 400 if any invalid.

    3. **src/lib/db/players.ts** -- Add one function:
       - `getAllPlayers(options?: { limit?: number; offset?: number })` -- returns all players ordered by `created_at` descending. This is needed by the admin panel later but should be added now as part of the player DB layer.

    4. **src/app/api/profile/avatar/route.ts** -- Already complete and correct. No changes needed. The existing code validates file type (image/*), size (5MB), uploads to Supabase Storage `avatars` bucket with upsert, and updates the player record.
