<task type="auto">
  <name>Task 2: Vercel deploy configuration and production deploy</name>
  <files>
    vercel.json
  </files>
  <action>
  1. Create `vercel.json` in project root with:
     ```json
     {
       "$schema": "https://openapi.vercel.sh/vercel.json",
       "framework": "nextjs",
       "headers": [
         {
           "source": "/sw.js",
           "headers": [
             { "key": "Cache-Control", "value": "no-cache, no-store, must-revalidate" },
             { "key": "Service-Worker-Allowed", "value": "/" }
           ]
         },
         {
           "source": "/manifest.json",
           "headers": [
             { "key": "Cache-Control", "value": "no-cache" }
           ]
         },
         {
           "source": "/(.*)",
           "headers": [
             { "key": "X-Content-Type-Options", "value": "nosniff" },
             { "key": "X-Frame-Options", "value": "DENY" },
             { "key": "Referrer-Policy", "value": "strict-origin-when-cross-origin" }
           ]
         }
       ]
     }
     ```

  2. Verify the build works:
     - Run `npm run build` to confirm everything compiles
     - Check that `public/sw.js` is generated by the build (Serwist output)

  3. Deploy to Vercel:
     - Check if Vercel CLI is available: `npx vercel --version`
     - Look for Vercel token: check environment variables (`echo $VERCEL_TOKEN`), then check 1Password (`op item list | grep -i vercel`)
     - Run: `npx vercel --prod --yes` (or with --token if needed)
     - If no Vercel token available, create the vercel.json and document the deploy command for the user

  4. After deploy, verify:
     - The production URL is accessible
     - `/manifest.json` returns correct JSON
     - `/sw.js` is served with correct headers

  AVOID: Do not set up a custom domain. The default .vercel.app URL is sufficient. Do not configure CI/CD pipelines. Do not modify environment variables on Vercel (Supabase env vars should already be in Vercel project settings or will be set by user).

  NOTE: If Vercel deploy fails due to missing env vars (NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY), document what needs to be set in Vercel project settings and proceed.
  </action>
  <verify>
  - `cat vercel.json | python3 -m json.tool` validates
  - `npm run build` passes
  - If deployed: production URL loads successfully, `/manifest.json` returns 200, `/sw.js` returns 200
  - If deploy blocked by auth: vercel.json exists and deploy command is documented
  </verify>
  <done>
  vercel.json configured with proper service worker and security headers. Build passes cleanly. App is deployed to Vercel (or deploy command documented if auth unavailable). Production URL serves the PWA correctly.
  </done>
