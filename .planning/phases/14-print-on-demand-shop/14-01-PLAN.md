# 14-01: Printify API Integration

## Task
Integrate Printify API for product sync, SKU mapping, and 40% markup pricing.

## Scope
Backend/data layer only. Shop UI changes are in 14-02-b.

## Implementation

### 1. Printify API Client (`src/lib/shop/printify.ts`)
- Server-side only client using Personal Access Token
- Endpoints: catalog, products, orders, shipping, webhooks
- Type-safe with TypeScript interfaces matching Printify API responses
- Rate-limit-aware with error handling

### 2. Product Types Enhancement (`src/lib/shop/products.ts`)
- Add `printifyProductId`, `printifyBlueprintId`, `printifyProviderId` fields
- Add `sku` field to variants
- Keep mock data as fallback when PRINTIFY_API_TOKEN not configured
- Pricing: `markup()` function applies 40% markup (already exists)

### 3. Product Sync Server Action (`src/lib/shop/sync.ts`)
- Fetch products from Printify API
- Map to local Product/Variant types
- Apply 40% markup to base costs
- Generate slugs from product titles
- SKU mapping from Printify variant IDs

### 4. API Routes
- `src/app/api/shop/products/route.ts` - Get synced products
- `src/app/api/shop/sync/route.ts` - Trigger product sync (admin only)
- `src/app/api/webhooks/printify/route.ts` - Handle Printify webhooks

### 5. Environment Variables
- `PRINTIFY_API_TOKEN` - Personal Access Token
- `PRINTIFY_SHOP_ID` - Shop identifier

## Design Decisions
- Keep existing mock products as fallback
- Server-side API calls only (no CORS from Printify)
- Products cached locally to avoid rate limits
- 40% markup applied via existing `markup()` function

## Success Criteria
1. Printify API client with full type definitions
2. Product sync logic with SKU mapping
3. 40% markup pricing calculation
4. API routes for product access and sync
5. Webhook handler for order updates
6. TypeScript compiles cleanly
