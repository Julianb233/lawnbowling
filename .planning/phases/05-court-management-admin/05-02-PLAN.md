## Objective
Verify and complete all admin panel features (ADMIN-01 through ADMIN-04). Fix identified gaps: venue sports management (ADMIN-03) and extended venue fields (ADMIN-01).

## Requirements Covered
- ADMIN-01: Admin can manage venue details (name, address, timezone)
- ADMIN-02: Admin can add/remove/edit courts and lanes
- ADMIN-03: Admin can add/remove sports available at venue
- ADMIN-04: Admin can view all players, waivers, match history

## Audit Findings

- `src/app/admin/venue/page.tsx`: Edits name, address, timezone -- works

- **GAP**: Venue type in `src/lib/types.ts` does not include extended fields
- **FIX**: Update Venue type and venue settings page to include contact info fields

### ADMIN-02 (Court CRUD) - COMPLETE
- `src/app/admin/courts/page.tsx`: Full CRUD UI (add, edit name+sport, delete, availability indicator)
- `src/app/api/admin/courts/route.ts`: GET, POST, PUT, DELETE handlers with admin auth

### ADMIN-03 (Sport Management) - MISSING
- DB schema has `venues.sports text[]` column
- **GAP**: No admin UI to manage available sports at venue
- **FIX**: Add sport management section to venue settings page (add/remove sports from venue.sports array)

- **FIX**: Update courts admin to use venue sports instead of hardcoded list

### ADMIN-04 (View Players/Waivers/History) - COMPLETE
- `src/app/admin/players/page.tsx`: Player list with search, skill filter, role toggle
- `src/app/admin/waivers/page.tsx` + AdminWaiversClient: Signed waivers list with search
