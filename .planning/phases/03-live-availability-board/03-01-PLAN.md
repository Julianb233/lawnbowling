## Remaining Work

### Task 1: Verify Supabase Realtime publication
- Confirm `alter publication supabase_realtime add table players;` is applied in the actual Supabase project
- If not applied, run migration or SQL statement in Supabase dashboard
- **Test:** Open two browser tabs, check in on one, verify the other sees the update without refresh

### Task 2: Add auto-checkout for stale sessions
- Players who close their browser stay "available" forever — no cleanup mechanism exists
- Add a Supabase Edge Function or cron job (via `pg_cron` or Next.js API route) to set `is_available=false` for players whose `checked_in_at` is older than 4 hours
- Create API route `src/app/api/cron/auto-checkout/route.ts` with Vercel cron config
- **Why:** Without this, the board fills with ghost players over time

### Task 3: Add check-in confirmation toast
- After toggling check-in/out, show a brief toast notification confirming the action
- Use existing UI toast component or add a lightweight one (e.g. `sonner`)
- **Why:** Currently the button state changes but there's no explicit confirmation feedback beyond the color change

### Task 4: Protect board page behind auth + waiver
- Add middleware or server-side check to `/board` route ensuring:
  1. User is authenticated (redirect to `/login` if not)
  2. User has a signed waiver (redirect to `/waiver` if not)
- Check existing middleware at `src/middleware.ts` for auth guards
- **Why:** AVAIL-01 implies only registered, waiver-signed players can check in
