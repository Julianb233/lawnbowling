## Remaining Work

### Task 1: iPad landscape kiosk optimization
- Current grid goes up to `xl:grid-cols-4` which is fine, but iPad landscape (1024px-1194px) hits the `lg:` breakpoint
- Verify the sidebar (MatchQueue + CourtStatusBoard at `w-72 lg:block xl:w-80`) doesn't crowd the board on iPad
- Test that player cards are readable and tap targets are adequate at iPad viewport
- May need to adjust the grid or sidebar width for 1024px specifically
- **Test at:** 1024x768 (iPad landscape), 1180x820 (iPad Air landscape), 1366x1024 (iPad Pro 12.9 landscape)

### Task 2: iPhone portrait optimization
- Current layout has `pb-20` for bottom nav padding and responsive grid starting at 1 column
- Verify player cards render well in single-column on small screens (375px, 390px, 428px widths)
- Check that the check-in button, filters, and cards all have 44px minimum touch targets
- The skill dropdown in `BoardFilters` may be too small on iPhone — verify tap size
- Bottom nav already has `min-h-[44px]` — good
- **Test at:** 375x812 (iPhone SE/13 mini), 390x844 (iPhone 14), 428x926 (iPhone 14 Plus)

### Task 3: Touch target audit
- Audit all interactive elements for minimum 44x44px touch targets per Apple HIG / WCAG:
  - `CheckInButton`: already large (`py-5 px-8`) — OK
  - `BoardFilters` sport buttons: `px-4 py-2` — may be borderline, increase to `py-2.5` if needed
  - `BoardFilters` skill dropdown: native `<select>` — browser handles touch area
  - `PlayerCard` "PICK ME" button: `py-2.5` — OK
  - `BottomNav` links: already `min-h-[44px]` — OK
- Fix any elements below 44px

### Task 4: Board empty state and loading polish
- Empty state exists with bouncing emoji — good
- Verify loading skeleton matches the actual board layout proportions
- Consider adding a subtle pulse animation to skeletons if not already present (check `skeleton` CSS class)
- Test that skeleton → real content transition is smooth (no layout shift)

### Task 5: Filter persistence (optional enhancement)
- Currently filters reset on page reload
- Consider saving sport/skill filter selections to URL search params (`?sport=pickleball&skill=intermediate`)
- This enables sharing filtered board links and preserves state on refresh
- Use `useSearchParams` + `useRouter` to sync state
