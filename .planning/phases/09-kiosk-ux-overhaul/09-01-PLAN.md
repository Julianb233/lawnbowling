# Plan 09-01: Kiosk Component Redesign

## Goal
Redesign all kiosk components for WCAG AAA accessibility targeting 60-80+ year old users.
Touch targets 56-72px, typography 16px minimum / 32px+ headings, 7:1 contrast ratio.

## Requirements
- KIOSK-01: Touch targets 56-72pt minimum
- KIOSK-02: WCAG AAA contrast (7:1 ratio)
- KIOSK-03: 4-screen check-in flow (Welcome -> Name Search -> Position -> Confirm)
- KIOSK-04: 15-second auto-reset after confirmation
- KIOSK-05: A-Z letter filter for 300 member capacity
- KIOSK-06: 10-second undo window

## Changes

### 1. KioskLayout.tsx — Redesign
- Use CSS custom properties from globals.css (--kiosk-*) instead of hardcoded values
- Add `kiosk-mode` class to root div for global kiosk styles
- Increase header font sizes (36px->40px heading)
- Add focus visible outlines (3px solid #0D47A1)
- Add reduced motion support
- Increase tab button sizes to 72px height
- Add time/date display in header

### 2. KioskCheckIn.tsx — Redesign
- Increase alphabet filter button sizes to 56px minimum
- Increase player row heights to 72px+
- Better contrast on all text elements
- Add focus management (auto-focus first player on load)
- Improved position selection with larger 88px+ buttons
- Confirmation screen with larger checkmark, clearer text
- Ensure all interactive elements have proper aria-labels

### 3. KioskWrapper.tsx — Improvements
- Larger "Are you still there?" overlay text
- Bigger continue button (72px+)
- Better contrast on overlay text

### 4. AdminPinModal.tsx — Redesign for Kiosk
- Larger keypad buttons (72px+)
- Higher contrast colors
- Larger PIN dots
- Use kiosk color palette instead of zinc
- Better aria labeling

### 5. Kiosk Pages — Add kiosk-mode class
- /kiosk/page.tsx — ensure kiosk-mode class applied
- /kiosk/scan/page.tsx — ensure consistent styling

## Verification
- `npm run build` passes
- All kiosk text >= 16px
- All touch targets >= 56px
- Color contrast >= 7:1 for text
