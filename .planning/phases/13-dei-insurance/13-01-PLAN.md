# Phase 13-01: In-App DEI Insurance Platform

## Objective
Build a lawn bowling-specific insurance product integrated into the app, with dedicated
pages, check-in integration, and admin coverage management.

## Deliverables

### 1. Insurance Landing Page (/insurance)
- General DEI partnership page explaining per-participant liability coverage
- Hero section with value proposition (free for venues, revenue per participant)
- How-it-works 3-step flow tied to check-in
- What's covered section (liability, medical, emergency transport, AD&D)
- GL blind spot educational section
- FAQ accordion
- Lawn bowls coverage callout linking to /insurance/lawn-bowls
- Trust badges (All 50 States, A-Rated Carriers, 500+ Facilities)

### 2. Lawn Bowls Insurance Page (/insurance/lawn-bowls)
- Lawn bowling-specific coverage page with injury statistics
- Three coverage tiers: Basic ($3), Standard ($8), Premium ($15)
- Injury stats: 54% falls, 35% fractures, 92% players 45+, $12K+ avg ER visit
- Coverage gap explainer (club GL vs individual coverage)
- Tournament/club integration section
- Lawn bowls-specific FAQ (age limits, away matches, club vs individual)
- Testimonial from tournament director
- SEO metadata with lawn bowling insurance keywords

### 3. Check-In Insurance Integration
- QR check-in (/checkin/[venueId]): Insurance offer appears after successful check-in
- Kiosk check-in (KioskCheckIn): Insurance upsell on confirmation screen
- Links to /insurance/lawn-bowls for coverage details

### 4. Admin Coverage View (/admin/insurance)
- Overview stats: total members, active coverage, expired, coverage rate percentage
- Searchable member list with insurance status filter
- Status badges: Active (green), Expired (amber), None (gray)
- Last check-in date display

### 5. Insurance Status API
- POST /api/insurance/status: Updates authenticated user's insurance_status
- Validates status values against InsuranceStatus type
- Returns updated status on success

### 6. Data Model
- InsuranceStatus type: "none" | "active" | "expired"
- insurance_status column on players table with check constraint
- Default value: "none"
