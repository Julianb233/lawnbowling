## Remaining Gaps

1. **Logout** — No explicit logout button/action found. Need to verify a logout mechanism exists somewhere in the UI (nav, settings, profile page).
2. **Admin seeding** — No seed script to create the first admin user. Need a script or SQL to promote a user to admin role.
3. **Schema deployment** — `schema.sql` exists but no migration runner or Supabase CLI config. Schema needs to be applied to actual Supabase project.
4. **Signup player profile mismatch** — Signup page inserts `{ id: data.user.id, name }` but schema has `user_id` (not `id` as the user_id). The signup page sets `id` to the auth user id, but the schema expects `user_id` as the FK. This is a potential bug — the `id` column is auto-generated UUID, and `user_id` is the FK to auth.users. The auth callback correctly doesn't set `id` but relies on `players.id` matching `auth.users.id`, which conflicts with the schema where `id` is a separate auto-generated UUID.
5. **Session persistence test** — Need to verify session survives browser refresh (middleware handles this, but untested).

## Steps

1. **Audit logout** — Check settings/profile pages for logout. If missing, add logout button to the app's navigation or settings page.
2. **Fix signup player insert** — The signup page (`src/app/signup/page.tsx` line 48) does `.insert({ id: data.user.id, name })` but should use `{ user_id: data.user.id, display_name: name }` per the schema. Same issue in auth callback (line 22-24).
3. **Create admin seed script** — Add `scripts/seed-admin.sql` or a Supabase function that promotes a user to admin by email.
4. **Add Supabase project config** — If not already present, document how to apply `schema.sql` (either via Supabase CLI migrations or manual SQL editor).
5. **Build and smoke test** — Verify the complete auth flow works end-to-end.
